1237. Find Positive Integer Solution for a Given Equation

问题

给定一个函数f(x, y)和数值z，返回所有的正整数对(x, y)满足$f(x, y) = z$

例如：

  • 1 <= function_id <= 9
  • 1 <= z <= 100
  • 1 <= x, y <= 1000
  • function是严格的单调增函数，满足
    • f(x, y) < f(x + 1, y)
    • f(x, y) < f(x, y + 1)

（注意题目不是让你去猜函数，而是通过某种搜索（？）根据函数调用结果看是否满足条件找到所有的解）

思路

因为函数是单调的增函数，可以给我们提供一些思路，也就是说，如果y从大到小，那么对应的满足条件的x应该是从小到大，那么我们可以在一个(x, y) （1000 * 1000）的二维解空间，从右上角x=1, y=1000开始，每次向下(小于目标值时）或者向左移动（大于等于目标值）。

因此时间复杂度为O(1000)

问题

给定一组文件名，例如’/leetcode/problems’，删除所有的子文件后返回剩下的文件名。（例如，存在‘/leetcode’ 和‘/leetcode/problems’, 那么后者是前者的子文件，删除后者）

例如：

思路

这题方法挺多的，给的时间和空间要求也很宽松，大概就是怎么写都能过==一个非常暴力的写法，遍历文件名的前缀文件名看是否存在母文件

这样写时间复杂度是$O(mn)$ (m是文件名数，n是平均文件名长度)，空间复杂度是也是$O(mn)$

  • 如果对文件名按字典序进行排序，那么一个文件所对应的所有的子文件一定紧跟在它之后。（考虑字典中的前缀对应的后缀紧跟在其后）

这样做相当于排序加一次遍历，时间复杂度为$O(n\log n)$

1235. Maximum Profit in Job Scheduling

问题

给定一组作业的开始时间和结束时间，利润，要求安排作业（时间段不能有交叉）使得利润最大化

例如：

思路

看上去就很像dp并且有点像01背包，但这里的一个问题是因为时间特别大，很显然的如果记录所有T会导致MLE，一个想法是记录index，然后转化回来，当然这样做就比较麻烦==

一个更加机智的思路[4]是维护一组这样的dp [[x, y]]，y表示占用时间到x时的最大利润，同时在维护过程中保证dp是有序的（按y排序），初始化为dp = [[0, 0]]，先贪心的对所有作业按结束时间排序（先结束的先做），然后每次考虑第i个作业时，最大利润为在dp中找到x=startTime[i]+1的最左边对应的y（可以用二分查找）+ profit[i]，可以理解为做第i个作业开始时间前的最大利润+第i个作业的利润。（能够这样保证、因为dp是有序的）

  • Q: 为什么$dp[i][1] < dp[-1]][1] $时不需要添加

    A: $dp[i][1] < dp[-1]][1]$是加入当前作业能得到的最大利润，因为作业已经按结束时间排序了，加入当前的作业之后结束时间必然会变长，如果结束时间变长而利润没有加大，因此没有必要加入当前的作业

1224. Maximum Equal Frequency

问题

给定一个数组，求最长的前缀数组的长度，要求前缀数组满足

  • 去掉一个数字后，数组内剩余所有数字的出现次数相同

例如，

输入 nums = [2,2,1,1,5,3,3,5]

输出 7 (对于前缀数组2,2,1,1,5,3,3，去掉5后，剩余所有数字出现的次数都为5)

思路

做的时候都没想到这题考查的是啥==就用了非常naive的一个做法，遍历数组，统计每个数字出现的次数count，然后统计出现次数的出现次数c_count（有点绕，比如对于2,2,1,1,5,3,3，count={2: 2, 1:2, 5: 1, 3:2}，c_count={2: 3, 1: 1}，c_count表示出现次数为2的数字有3个，出现次数为3的数字有1个）那么符合要求的数组有如下情况：

  • len(count) == 1，只有一个数字出现，显然满足
  • len(c_count) == 1
  • len(c_count) == 2，表示数字出现次数有两个值，那么
    • 要么其中一个的k, v都为1，表示出现次数为1的数字有1个，那么将其删掉，剩下所有数字的出现次数都相同
    • 假设c_count = {k1: v1, k2: v2}，要求k1 = k2+1, v1 = 1 或者k2 = k1+1, v2 = 1，也就是说有两种出现次数，大的出现次数只比少的出现次数大1，且只有一个这样的数字，这样删去一个这样的数字即可

看到一个非常简洁的代码，思路其实差不多，维护两个dict，count和c_count，从c_count的maxf考虑 - maxf表示数字出现次数的最大值，符合条件的前缀数组arr[:i]满足

  • 所有数字都只出现一次，那么maxf = 1

  • 所有数字都出现maxf次，除了有一个数字出现一次，那么c_count[maxf] * maxf + 1 = i + 1

  • 所有数字都出现maxf-1次，除了有一个数字出现maxi次，那么

    ​ $c_count[maxf-1] * (maxf-1) + maxf= i + 1$

1220. Count Vowels Permutation

问题

计算元音字符串的排列个数。给定一个正整数n，假设需要组成一个长度为n的字符串，要求：

  • 全部由小写的元音字母a, e, i, o, u组成
  • ‘a’后面只能接着’e’
  • ’e’后面只能接着’a’或者’i’
  • ‘i’后面只能接着不是’i’的字母
  • ‘o’后面只能接着‘i’或者’u’
  • ‘u‘后面只能接着’a’

返回可能的排列次数mod 10^9 + 7.

例如，

输入 n=1，输出5

输入 n=2，输出10

输入 n=5，输出68

思路

dfs

最直接的一个思路就是dfs，按follow数组往下搜

（这样的时间复杂度是O(5^n)，预料之中的TLE了）

dp

在手动模拟这个树的遍历的时候，可以观察到，假设用dp[n][j]表示字符串长度为n时以字符j结尾的个数，那么

$dp[n][j] = sum(dp[n-1][k])$, if (k, j) exist​

因此用一个pre_idx[i]记录所有可以指向i的字符index（按字典序的index)，然后因为动态规划时只需要用到上一个状态，因此只需要一个pre数组存储之前的状态即可，不需要二维数组。

（时间复杂度为O(log n)）

Intro

LC 128. Longest Consecutive Sequence

题目

给定一个无序数组，返回包含的最长连续序列的长度

例如：

输入：[100, 4, 200, 1, 3, 2]

输出：4 （包含[1, 2, 3, 4]

思路

首先建一个数组{value, idx}的映射mm，对于数组的元素arr[i]=x，建立一个以x的下表i为索引的并查集，如果存在相邻的元素x-1/x+1，那么合并i和mm[x-1]/mm[x+1]，同时并查集中加入一个size记录集合的大小，union时按size合并，最后返回size的最大值。因为find和union操作近似常数时间复杂度，因此总的时间复杂度为O(n)。

1208. Get Equal Substrings Within Budget

问题

给定两个长度相同的字符串s, t，可以将s[i]转换为t[i]，代价为$|s[i]- t[i]|$，最大代价为maxCost，返回能够s转化为t的substring（连续的子字符串）的最大长度。

输入：s = “abcd”, t = “bcdf”, maxCost = 3
输出：3

思路

two pointer问题，用左右两个指针维护一个滑动窗口，计算滑动窗口内的转换代价，left=0, right从0移动到len(s)-1，如果超过了maxCost, left+=1。
最后返回滑动窗口的长度（right-left+1)。

注意最后返回的left:right并不是最长的转换字符串，但可以这么考虑，在我们的遍历过程中，滑动窗口的长度只会增加不会减小，因此最后的长度等于最长的满足要求的子字符串长度。

1202. Smallest String With Swaps

问题

给定一个字符串和一个二元索引数组，假设这些二元索引对应的字符可以交换任意次，求交换能够得到的最小字典序的字符。

  • 在同一连通分量内，对索引对应的字符排序，然后按索引的顺序从小到大赋值

1203. Sort Items by Groups Respecting Dependencies

问题

给定n个item，属于m个group，group[n]表示item对应的group，为-1时表示不属于任何group，要求属于同一个group的items连续。beforeItems[n]表示item要求
在其之前的items，也就是说对于i，beforeItems[i]中的元素必须要i之前。返回满足上述两个要求的item的一个排列。

输入：n = 8, m = 2, group = [-1,-1,1,0,0,1,0,-1], beforeItems = [[],[6],[5],[6],[3,6],[],[],[]]
输出：[6,3,4,1,5,2,0,7]

思路

典型的拓扑排序（入度为0时删除节点加入序列，并将节点指向的点入度减一，最后得到的序列如果和节点数相等，那么就得到一个拓扑排序，否则不存在），加上group的要求之后，可以考虑分两个拓扑排序：1）对于group之间的依赖关系；2）group内的依赖关系。

那么具体思路为：

  • 根据group，构建group和items的关系。（为-1时为单独一个group）
  • 根据beforeItems构建group间和group内部的依赖关系（图，graph[i][j]和to[j]）
  • 两个拓扑排序，根据group间的拓扑顺序，进行group内的拓扑排序）

（写起来挺烦的，注意：beforeItems[i]增加的是i的入度，做item的拓扑排序需要根据group拓扑排序的顺序，每次做完拓扑排序时检查序列大小）
还有一个做法，给每个group加源点和会点，对于不同组的依赖加在源点和会点上，这样就可以不用做两次拓扑排序（然而写起来也没有简单多少==）

注意：这两种写法实际上在构造group和group之间的依赖关系时其实都有可能构造重复的边，但是对拓扑来说没关系，因为在出i时会将i指向的所有边的删除，所以有重边对拓扑排序是没有影响的。
